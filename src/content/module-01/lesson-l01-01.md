## 什么是大语言模型

大语言模型（Large Language Model，LLM）本质上是一个**超大规模的"下一个 token 预测器"**。你给它一段文本，它预测下一个最可能出现的 token，然后把预测结果拼回去，继续预测下一个——如此循环，直到生成完整的回复。

### Token 是什么

Token 是 LLM 处理文本的最小单位。它不是"字"也不是"单词"，而是介于两者之间的"子词"（subword）。

一个粗略的经验法则：
- 英文：1 token ≈ 0.75 个单词 ≈ 4 个字符
- 中文：1 token ≈ 1.5-2 个汉字

### 上下文窗口

上下文窗口（Context Window）是模型一次能"看到"的最大 token 数量。它决定了：
- 你能塞多少背景信息
- 对话能持续多长
- Agent 能记住多少历史步骤

主流模型的上下文窗口（2025-2026）：
- Claude 3.5 Sonnet / Opus 4: 200K tokens
- GPT-4o: 128K tokens
- Gemini 2.5 Pro: 1M tokens

**工程约束**：窗口越大，推理越慢、成本越高。不要因为"窗口够大"就无脑塞内容——长上下文中的信息检索准确率会随位置变化（"Lost in the Middle"效应）。

### 为什么 LLM 会"幻觉"

幻觉（Hallucination）的根源在于 LLM 的工作方式——它不是在"查找事实"，而是在"预测概率最高的下一个 token"。

```
用户：巴黎是哪个国家的首都？
模型：法（概率最高）→ 国（概率最高）→ 。（概率最高）
```

这个过程中，模型没有"查数据库"，它只是根据训练数据中的统计模式来生成。当它遇到训练数据中信息不足或相互矛盾的问题时，就会"编造"一个看起来合理但实际错误的答案。

**降低幻觉的工程手段**：
- RAG（检索增强生成）：先查到真实信息，再让模型基于信息回答
- Function Calling：让模型调用工具获取实时数据
- 结构化输出：约束输出格式，减少自由发挥空间

### 训练、微调、推理三个阶段

```
预训练（Pre-training）    →  微调（Fine-tuning）    →  推理（Inference）
海量文本 → 基础模型        标注数据 → 对齐模型        用户输入 → 实时输出
数十亿美元                数千到数百万美元            按 token 计费
```

大多数 Agent 开发者只需要关注**推理**阶段——如何用好已有的模型。但理解前两个阶段能帮你更好地理解模型的"性格"和"边界"。

### 要点总结

- LLM 的本质是 token 预测器，不是数据库
- Token 是子词单位，中英文的 token 化效率不同
- 上下文窗口是 Agent 记忆的硬上限
- 幻觉来自概率预测机制，需要通过工程手段抑制
- 作为开发者，你主要关注推理阶段，但理解训练和微调有助于模型选型
